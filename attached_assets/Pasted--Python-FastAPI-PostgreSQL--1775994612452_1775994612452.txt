Технологический стек
Язык: Python
Фреймворк: FastAPI (асинхронный)
Базы данных: PostgreSQL (отдельный инстанс для каждого микросервиса)
Аутентификация: JWT
Тестирование: pytest
Линтеры: flake8, mypy
Форматтер: black
Git hooks: pre-commit (запуск линтеров и тестов)
Контейнеризация: Docker, docker-compose
Никита Сычев – отвечает за сервис пользователей и колод (регистрация, JWT, управление колодами), а также за API Gateway (единая точка входа, проверка токенов, маршрутизация).
Полина Васильева – разрабатывает сервис каталога карт (CRUD карт, фильтрация, загрузка начальных данных) и предоставляет данные о картах другим сервисам.
Аня Смирнова – реализует сервис игровой логики (создание матчей, механика ходов, подсчёт очков, определение победителя) и взаимодействует с сервисами колод и карт для получения данных.






Микросервисы и распределение обязанностей

Сервис
Ответственный
Краткое описание
User & Deck Service
Никита
Управление пользователями (регистрация, аутентификация, JWT). Управление колодами (CRUD, добавление/удаление карт, валидация принадлежности колоды пользователю).
Card Catalog Service
Полина
Каталог карт (CRUD, фильтрация по параметрам, получение по ID). Загрузка начальных данных (набор карт). Предоставление информации о картах для других сервисов.
Game Service
Аня
Логика матчей: создание игры, совершение ходов, подсчёт очков, определение победителя, завершение игры. Взаимодействие с User/Deck и Card сервисами для проверки колод и получения свойств карт.
API Gateway
Никита
Единая точка входа для клиентов. Проверка JWT-токенов. Маршрутизация запросов к соответствующим микросервисам. Обеспечение сквозной аутентификации.

Задачи
Никита
 Создать структуру сервиса user_deck.
– Определить модели User (id, username, hashed_password) и Deck (id, user_id, name).
– Реализовать эндпоинты:
   POST /register (создание пользователя),
   POST /login (выдача JWT),
   GET /profile (текущий пользователь).
– Для Deck: POST /decks, GET /decks, GET /decks/{id}, DELETE /decks/{id} (пока без связи с картами).
– Написать unit-тесты для этих эндпоинтов.
Полина
– Создать структуру сервиса card.
– Определить модель Card (id, name, strength, faction, type, description).
– Реализовать эндпоинты:
   GET /cards (список с фильтрацией по параметрам),
   GET /cards/{id} (детали карты),
   POST /cards (для наполнения каталога).
– Загрузить начальные данные (минимум 10 карт через миграцию или seed).
– Написать unit-тесты.
Аня
– Создать структуру сервиса game.
– Определить модели Game (id, player1_id, player2_id, player1_deck_id, player2_deck_id, state, current_turn, score1, score2).
– Реализовать эндпоинты:
   POST /games (создание игры, принимает id игроков и id колод),
   GET /games/{id} (получение состояния игры).
– Написать unit-тесты для создания и чтения игры.
